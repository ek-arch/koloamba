### Known limitation of the current snapshot approach

`kolo_balances` is a **point-in-time snapshot** from 2026-04-17. Balances drift as users keep tapping in the mini-app. Options for keeping it fresh:
- Re-import the same way periodically (manual, labor-intensive)
- Eventually wire to a live read-replica (the proper Phase 8 implementation)
- Expose a small internal API from the mini-app backend that this app can query per-user
